## Filter Word Alternatives

Instead of filter words, USE:

| 금지 | 대안 |
|------|------|
| 느꼈다 | 몸이 반응하는 묘사 (심장이 뛰었다, 손이 떨렸다) |
| 보였다 | 직접 시각 묘사 (빛이 쏟아졌다, 그림자가 드리웠다) |
| 생각했다 | 내면 독백 직접 서술 (이건 아닌데.) |
| 깨달았다 | 깨달음의 순간을 장면으로 (퍼즐 조각이 맞춰졌다) |
| ~것 같았다 | 단정적 서술 (그녀는 화가 났다) |

## Transition Gap Detection

Check each scene boundary for:

| Gap Type | Indicators | Bridge Example |
|----------|-----------|----------------|
| Temporal | Different time of day/date | "한 시간 후," / "다음 날 아침," |
| Spatial | Different location | "카페를 나서자 차가운 바람이 얼굴을 때렸다." |
| Emotional | exit_emotion != next entry_emotion | Show the shift through brief action/thought |

## When to Bridge

Add bridge ONLY when:
- 2+ gap types detected between scenes
- Reader would be confused without context
- Pacing requires breathing room

Do NOT bridge when:
- Scenes flow naturally
- Jump cut is intentional (cliffhanger effect)
- Only 1 minor gap type

# write-scene Skill

## Overview

장면 기반 집필 파이프라인을 실행합니다:
1. 챕터를 SceneV5 장면들로 분해
2. 각 장면별 스타일 예시 주입
3. scene-drafter 에이전트로 장면 집필
4. assembly-agent로 장면 조합
5. **revision loop 실행**: quality-oracle 분석 -> prose-surgeon 수술 -> 재분석 (최대 3회)
6. 최종 품질 검증 및 저장

### Step 4: Revision Loop (CRITICAL ORCHESTRATION)

**This is where Quality Oracle -> Prose Surgeon orchestration happens.**

```typescript
import { runRevisionLoop } from './src/pipeline/revision-loop.js';

// Define the surgeon callback that invokes prose-surgeon agent
const surgeonCallback = async (directive, prompt, modelConfig) => {
  // 1. Build agent invocation with directive and prompt
  // 2. Call prose-surgeon agent with appropriate model
  // 3. Return the fixed text from agent
  return await invokeProseSurgeonAgent(directive, prompt, modelConfig);
};

// Run the revision loop
const revisionResult = await runRevisionLoop(
  assembledChapter,
  scenes,
  surgeonCallback,
  {
    maxIterations: 3,           // Default: 3 Oracle->Surgeon cycles
    maxDirectivesPerPass: 5,    // Quality Oracle max directives
    stopOnCircuitBreak: true,   // Stop if same directive fails 3x
    verboseLogging: true,
  },
  styleLibrary
);

// Handle result
switch (revisionResult.finalVerdict) {
  case 'PASS':
    console.log('Quality check passed!');
    break;
  case 'REVISE':
    console.warn('Max iterations reached, some issues remain');
    break;
  case 'CIRCUIT_BREAK':
    console.error('Circuit breaker triggered:', revisionResult.circuitBreakReason);
    // Escalate to user for manual intervention
    break;
}
```

```

         ▼

│ For each        │

│                 │
│  ┌────────────┐ │
│  │ Prose      │ │
│  │ Surgeon    │ │
│  │ (callback) │ │
│  └────────────┘ │
│                 │
│ Apply fixes     │

         │
         ▼

│ iteration++     │

```
