# /design-sub-arc - 서브아크 설계

[NOVEL-SISYPHUS: 서브아크 설계]

$ARGUMENTS

## Idempotency

If output file already exists:
1. Read existing content
2. Preserve user modifications (fields not auto-generated)
3. Merge with new content
4. Warn if conflicting changes detected

## 실행 단계

1. **인자 확인**
   - 아크명 지정 시: 해당 서브플롯만 설계
   - 미지정 시: 필요한 서브플롯 제안 후 설계

2. **plot-architect 에이전트 호출**
   ```
   Task(subagent_type="plot-architect", prompt="
   메인 아크: {main-arc.json}
   캐릭터: {characters}

   서브플롯을 상세히 설계해주세요:

   **필수 항목 (상세 작성):**
   - 서브플롯 ID, 이름
   - **설명 (description)**: 서브플롯의 핵심 갈등, 전개 방향, 결말 방향을 500자 이상으로 상세히
   - 관련 캐릭터와 각 캐릭터의 역할
   - **메인플롯 연결점 (connection_to_main)**: 메인 스토리와 어떻게 상호작용하는지 300자 이상
   - 시작 회차, 해결 회차
   - **테마적 기능 (thematic_function)**: 작품의 주제를 어떻게 강화/대조하는지 200자 이상
   - **비트 (beats)**: 각 비트의 event는 해당 회차에서 일어나는 구체적 사건과 감정 변화를 200자 이상으로

   서브플롯 예시:
   - 직장 내 경쟁자와의 갈등 (승진, 프로젝트 경쟁)
   - 가족과의 화해 (과거 상처, 오해 해소)
   - 전 연인의 재등장 (미련, 비교, 성장)
   ")
   ```

3. **파일 생성**
   - `plot/sub-arcs/{arc_id}.json`
