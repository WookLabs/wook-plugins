# /blueprint-review

BLUEPRINT.md를 3개 관점에서 검토하고 개선합니다.

## 검토 관점

1. **구조** (critic): 플롯, 캐릭터 아크, 갈등
2. **상업성** (beta-reader): 훅, 몰입, 트렌드
3. **장르** (genre-validator): 필수 요소, 클리셰

## 사용법

```bash
/blueprint-review
/blueprint-review novels/my-novel/BLUEPRINT.md
```

## 워크플로우

### 1. 블루프린트 로드
- 현재 프로젝트의 BLUEPRINT.md 또는 지정된 경로

### 2. 병렬 검토
3개 에이전트가 동시에 검토:
- **critic**: 3막 구조 균형, 갈등 강도, 캐릭터 아크
- **beta-reader**: 독자 후킹, 트렌드 적합성, 이탈 위험
- **genre-validator**: 필수 트로프, 장르 기대 충족

### 3. 결과 종합
```
종합 점수: {N}/100

| 영역 | 점수 | 상태 |
|------|------|------|
| 구조 | 85/100 | ✅ |
| 상업성 | 72/100 | ⚠️ |
| 장르 | 90/100 | ✅ |
```

### 4. 수정 옵션
- "수정" - 권장사항 자동 반영
- "직접 수정" - 수동 수정
- "무시" - 현재 상태로 진행

## 검토 체크리스트

### 필수 요소
- 로그라인 (한 문장)
- 시놉시스 (200자/1000자)
- 주인공 Want/Need
- 3막 전환점
- 필수 트로프

### 상업성
- 1화 훅 강도
- 사이다 요소 (3-5화 내)
- 독자 몰입 요소

### 일관성
- 캐릭터-플롯 일치
- 세계관-플롯 지원
- 톤-장르 적합

## 옵션

| 옵션 | 설명 |
|------|------|
| `--auto-fix` | 피드백 자동 반영 |
| `--strict` | 엄격한 기준 |
| `--focus=AREA` | 특정 영역만 (structure/commercial/genre) |

## 출력

- 종합 점수 (0-100)
- 강점/약점 분석
- 구체적 개선안
- 자동 수정 옵션
