# Wisdom File Format Guide

각 wisdom 파일의 상세 형식과 규칙을 정의합니다.

## style-discoveries.md

문체와 서술 기법에 대한 발견사항을 기록합니다.

### 형식

```markdown
# Style Discoveries

## [챕터 N] 발견 제목
- **효과적**: 어떤 기법이 왜 효과적이었는지 서술
- **적용**: 향후 어떤 상황에서 활용할 수 있는지

## [챕터 N] 또 다른 발견
- **효과적**: ...
- **적용**: ...
```

### 규칙
- 챕터 번호는 해당 기법이 발견된 챕터를 기록
- 하나의 챕터에서 여러 발견이 나올 수 있음
- 발견 제목은 기법의 핵심을 간결하게 요약
- **효과적** 필드는 기법의 효과와 이유를 구체적으로 서술
- **적용** 필드는 재사용 가능한 상황을 명시

### 예시

```markdown
# Style Discoveries

## [챕터 3] 비 내리는 장면의 감각 묘사
- **효과적**: 빗소리를 대화의 배경으로 깔면서 인물의 내면 독백과 교차시킴. 빗방울이 유리창을 타고 흐르는 묘사가 눈물의 대리자 역할을 하여 직접적 감정 서술 없이 슬픔을 전달
- **적용**: 캐릭터가 감정을 억누르는 장면에서 자연 현상을 감정의 대리 표현으로 활용

## [챕터 7] 짧은 문장의 긴장감 조성
- **효과적**: 추격 장면에서 문장 길이를 3-5어절로 줄이고 한 줄 문단을 사용. 시각적으로도 속도감이 느껴지며 독자의 호흡이 빨라지는 효과
- **적용**: 액션/추격/위기 장면에서 문장 호흡을 의도적으로 짧게 조절

## [챕터 12] 시점 전환을 통한 서스펜스
- **효과적**: 주인공이 위기에 처한 순간 다른 캐릭터 시점으로 전환하여 독자의 궁금증 극대화. 전환된 시점의 캐릭터가 주인공의 위기를 모르고 일상을 보내는 대비 효과
- **적용**: 클리프행어 직후 또는 결정적 순간에 시점 전환으로 서스펜스 연장
```

## voice-patterns.md

캐릭터별 말투와 어조 패턴을 기록합니다.

### 형식

```markdown
# Voice Patterns

## 캐릭터명
- **기본 어조**: 반말/존댓말/혼합/기타
- **특징적 표현**: "자주 쓰는 말버릇이나 표현"
- **감정 변화 패턴**: 화날 때 / 슬플 때 / 기쁠 때 / 긴장할 때
- **첫 등장**: 챕터 N
- **비고**: 추가 특이사항
```

### 규칙
- 캐릭터명은 `characters/*.json`의 이름과 일치시킴
- 하나의 캐릭터에 하나의 섹션 (중복 금지)
- 새 패턴 발견 시 기존 섹션에 추가/갱신
- 감정 변화 패턴은 최소 2가지 이상 기록
- 특징적 표현은 큰따옴표로 감싸서 실제 대사 예시 포함

### 예시

```markdown
# Voice Patterns

## 강민호
- **기본 어조**: 반말 (친한 사이), 격식체 (공식 상황)
- **특징적 표현**: "그게 말이 되냐?", 혼잣말 시 "...씨" (욕설 참는 버릇)
- **감정 변화 패턴**:
  - 화날 때: 문장이 짧아지고 끊김. "됐어. 나가."
  - 슬플 때: 평소보다 말이 많아지며 주제를 돌림
  - 긴장할 때: 반말에서 존댓말로 전환 "그... 그렇습니까?"
- **첫 등장**: 챕터 1
- **비고**: 거짓말할 때 코를 만지는 습관 (서술에 반영)

## 이서연
- **기본 어조**: 존댓말 기본이나 독백에서는 반말
- **특징적 표현**: "알겠습니다만..." (반대 의견 제시 전 습관적 수긍)
- **감정 변화 패턴**:
  - 화날 때: 오히려 말이 더 정중해짐. 차가운 공손함
  - 슬플 때: 말이 줄고 단답
  - 기쁠 때: 독백에서 "오예!" 같은 구어체 등장
- **첫 등장**: 챕터 2
- **비고**: 전문 용어를 자연스럽게 섞어 씀 (의사 직업 반영)
```

## plot-threads.md

복선과 떡밥의 추적 상태를 기록합니다.

### 형식

```markdown
# Plot Threads

## [STATUS] 복선 제목
- **심은 위치**: 챕터 N, "관련 문장 또는 요약"
- **회수 예정**: 챕터 M
- **상태**: PLANTED / DEVELOPING / RESOLVED
- **메모**: 회수 방향성이나 주의사항
```

### 상태 코드

| 상태 | 의미 | 헤더 표시 |
|------|------|----------|
| `PLANTED` | 복선이 심어졌으나 아직 언급되지 않음 | `[ACTIVE]` |
| `DEVELOPING` | 복선이 간접적으로 언급/발전되고 있음 | `[ACTIVE]` |
| `RESOLVED` | 복선이 완전히 회수됨 | `[RESOLVED]` |

### 규칙
- 헤더의 `[STATUS]`는 `[ACTIVE]` 또는 `[RESOLVED]`만 사용
- PLANTED와 DEVELOPING은 모두 `[ACTIVE]`로 표시, 상세 상태는 **상태** 필드에 기록
- 회수 완료 시 헤더를 `[RESOLVED]`로 변경하고 회수 위치를 메모에 추가
- 심은 위치의 인용문은 원문에서 핵심 문장을 발췌
- 회수 예정 챕터는 플롯 설계 기준이며 유동적일 수 있음

### 예시

```markdown
# Plot Threads

## [ACTIVE] 검은 반지
- **심은 위치**: 챕터 3, "할머니가 건넨 검은 반지는 유난히 차가웠다"
- **회수 예정**: 챕터 12
- **상태**: PLANTED
- **메모**: 반지의 정체는 영혼결속의 매개체. 12화에서 위기 상황에 빛나기 시작

## [ACTIVE] 지하실 비밀문
- **심은 위치**: 챕터 7, "벽지 뒤로 미세한 틈이 보였지만 민호는 무시했다"
- **회수 예정**: 챕터 15
- **상태**: DEVELOPING
- **메모**: 10화에서 서연이 "이 집 구조가 이상해"라고 언급하며 간접 발전

## [RESOLVED] 엄마의 편지
- **심은 위치**: 챕터 1, "서랍 깊숙이 밀어 넣은 봉투를 또 꺼내다 넣었다"
- **회수 예정**: 챕터 8
- **상태**: RESOLVED
- **메모**: 챕터 9에서 회수. 편지 내용이 아버지의 비밀과 연결되어 메인 아크 전환점
```

## terminology.md

확정된 고유명사와 설정 용어를 기록합니다.

### 형식

```markdown
# Terminology

## 용어
- **정의**: 설명
- **첫 등장**: 챕터 N
- **관련 설정**: 연관된 세계관 요소나 다른 용어
```

### 규칙
- 용어는 가나다순/알파벳순 정렬 권장 (필수는 아님)
- 동음이의어는 별도 섹션으로 분리
- 정의는 작품 세계관 내의 의미를 기준으로 작성
- 첫 등장 챕터는 해당 용어가 본문에서 처음 사용된 챕터
- 관련 설정은 `world/world.json` 또는 다른 설정 파일과의 연결점

### 예시

```markdown
# Terminology

## 그림자 보행
- **정의**: 그림자를 매개로 공간을 이동하는 능력. 어둠이 깊을수록 이동 거리가 늘어나지만 정신력을 크게 소모한다
- **첫 등장**: 챕터 4
- **관련 설정**: 그림자 마법 체계, 민호의 고유 능력

## 영혼결속
- **정의**: 두 존재의 생명력을 영구적으로 연결하는 금지된 마법. 한쪽이 죽으면 다른 쪽도 함께 죽는다
- **첫 등장**: 챕터 5
- **관련 설정**: 검은 반지 (매개체), 고대 마법 금기 목록

## 황혼의 문
- **정의**: 현실 세계와 그림자 세계를 잇는 관문. 해 질 무렵에만 열린다
- **첫 등장**: 챕터 8
- **관련 설정**: 세계관 지리 (그림자 세계), 지하실 비밀문과 연결
```

## lessons.md

집필 과정에서 얻은 교훈과 기법을 기록합니다.

### 형식

```markdown
# Lessons Learned

## [챕터 N] 교훈 제목
- **발견**: 무엇을 깨달았는지
- **효과적**: 잘 작동한 점
- **비효과적**: 잘 작동하지 않은 점
- **향후 적용**: 앞으로 어떻게 활용할지
```

### 규칙
- 챕터 번호는 해당 교훈을 얻은 챕터
- **효과적**과 **비효과적** 중 하나만 해당되어도 기록 가능 (비해당 필드는 "N/A" 처리)
- **향후 적용**은 구체적이고 실행 가능한 지침으로 작성
- 동일한 교훈이 반복 확인되면 기존 항목에 "재확인: 챕터 M" 추가

### 예시

```markdown
# Lessons Learned

## [챕터 5] 회상 장면의 길이 조절
- **발견**: 회상이 3페이지를 넘으면 현재 서사의 긴장감이 급격히 떨어진다
- **효과적**: 2페이지 이내의 회상은 캐릭터 깊이를 더하면서 흐름을 유지
- **비효과적**: 5페이지짜리 회상 → 퇴고 시 editor가 "페이싱 붕괴" 지적
- **향후 적용**: 회상 장면은 2페이지 이내로 제한. 긴 회상이 필요하면 현재 서사와 교차 편집

## [챕터 9] 대화 중 행동 묘사 삽입
- **발견**: 긴 대화 장면에서 인물의 소소한 행동을 사이사이 넣으면 장면이 살아남
- **효과적**: "컵을 돌리며" "시선을 피하며" 같은 짧은 행동 태그가 캐릭터성 강화
- **비효과적**: N/A
- **향후 적용**: 대사 5줄 이상 연속되면 반드시 행동/묘사 태그 삽입. 행동은 감정 상태를 간접 반영

## [챕터 14] 장르 전환 포인트의 톤 관리
- **발견**: 일상 파트에서 미스터리 파트로 전환할 때 갑작스러운 톤 변화가 독자를 당황시킴
- **효과적**: 전환 전 1-2문단에 미묘한 불안 요소를 배치하면 자연스러운 전환
- **비효과적**: 직전 문단까지 코믹한 톤 → 갑자기 공포 묘사 (재확인: 챕터 18)
- **향후 적용**: 톤 전환 시 "전환 브릿지" 문단을 반드시 삽입. 일상 속 미세한 위화감으로 시작
```

## 공통 규칙

1. **인코딩**: UTF-8
2. **줄바꿈**: 항목 간 빈 줄 하나
3. **헤더 레벨**: 파일 제목은 `#`, 각 항목은 `##`, 하위 정보는 리스트(`-`)
4. **날짜 기록**: 필요 시 항목 끝에 `(추가: YYYY-MM-DD)` 형식으로 기록
5. **수정 이력**: 기존 항목 수정 시 원본을 유지하고 아래에 `> 수정 (챕터 N): 내용` 추가
