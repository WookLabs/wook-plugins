# Review Skill - Example Usage

## Example 1: 세계관 검토

### 사용자 요청
```
사용자: 세계관 설계 완료했어. 검토해줘.
```

### 스킬 실행
```markdown
# 세계관 설계 검토 결과

## 최종 판정: REVISE

**종합 점수**: 73/100
**신뢰도**: HIGH (편차: 8)

---

## 리뷰 요약

| 관점 | 점수 | 판정 | 주요 이슈 |
|------|------|------|-----------|
| 기술적 품질 (critic) | 75/100 | REVISE | 마법 체계의 제약이 불명확 |
| 독자 경험 (beta-reader) | 78/100 | ENGAGING | 세계관은 매력적이나 설명이 과다 |
| 장르 적합성 (genre-validator) | 65/100 | ACCEPTABLE | 필수 트로프 일부 누락 |

---

## 상세 피드백

### ✅ 강점 (Strengths)

#### 기술적 측면
1. **사회 계층 구조가 상세하고 논리적**
   - 위치: `world/world.json:social_structure`
   - 귀족 5계급, 평민 3계급이 명확히 정의됨
   - 계층 간 이동 규칙도 설명됨

2. **지리적 설정이 플롯과 잘 연동**
   - 위치: `world/world.json:geography`
   - 북부/남부 기후 차이가 주인공 여정에 활용됨
   - 수도/변경 지역 구분이 정치 플롯 지지

3. **용어가 일관적으로 사용**
   - 마법사: "메이지"로 통일
   - 화폐: "크라운"으로 통일
   - 용어집이 잘 정리됨

#### 독자 경험 측면
1. **독특한 마법 체계가 호기심 자극**
   - 위치: `world/world.json:magic_system`
   - "감정 기반 마법"이라는 신선한 설정
   - 독자가 "어떻게 작동하지?" 궁금해할 것

2. **세계관이 시각적으로 풍부**
   - 수정 궁전, 하늘 섬 등 이미지화하기 좋은 요소
   - 독자 몰입도 높을 것으로 예상

#### 장르 측면
1. **"금지된 사랑" 트로프 잘 활용**
   - 귀족-평민 계급 차이가 로맨스 장애물로 작동
   - 판타지 로맨스 독자가 좋아하는 설정

---

### ⚠️ 개선 필요 (Needs Revision)

#### 우선순위 높음

1. **마법 체계의 제약이 불명확** (critic 지적)
   - 발견자: critic
   - 위치: `world/world.json:magic_system.limitations`
   - 현재 내용:
     ```json
     {
       "limitations": "감정이 불안정하면 마법 실패 가능"
     }
     ```
   - 문제: 너무 모호함. "불안정"의 기준은? "실패"의 결과는?
   - 영향:
     - 플롯 긴장감 저하 (주인공이 언제든 마법으로 해결 가능)
     - 데우스 엑스 마키나 위험 (편리하게 마법 사용)
   - 제안:
     구체적 제약 3가지 명시:
     ```json
     {
       "limitations": {
         "mana_cost": "감정 소모량: 슬픔(50%), 분노(30%), 기쁨(20%)",
         "cooldown": "같은 감정 재사용: 1시간 대기",
         "side_effect": "과도한 사용 시 감정 둔화 (3일간 무감정 상태)"
       }
     }
     ```

2. **필수 트로프 누락** (genre-validator 지적)
   - 발견자: genre-validator
   - 위치: `world/world.json`
   - 문제: 판타지 로맨스 필수 요소인 "운명적 연결"이 세계관에 없음
   - 영향: 장르 독자 기대 미충족, "그냥 만난 거네" 느낌
   - 제안:
     세계관에 "소울메이트" 또는 이에 준하는 설정 추가:
     - 옵션 1: 마법적 유대 (영혼의 공명)
     - 옵션 2: 전생 인연 (과거 생애 연결)
     - 옵션 3: 운명의 표식 (태어날 때부터 정해진 인연)

#### 우선순위 중간

3. **세계관 설명이 과다** (beta-reader 지적)
   - 발견자: beta-reader
   - 위치: `world/world.json` 전반
   - 문제: 설명이 너무 상세해서 독자가 압도될 수 있음
   - 예시:
     - 역사 섹션이 3000단어 (독자가 읽기 부담)
     - 계층 구조가 8단계 (복잡함)
   - 영향: 독자가 "세계관이 복잡하다", "이해 안 된다" 느낄 위험
   - 제안:
     - 핵심 요소만 남기고 나머지는 작중에서 자연스럽게 공개
     - 역사: 3000단어 → 500단어로 축소 (핵심 사건만)
     - 계층: 8단계 → 5단계로 단순화

4. **용어 불일치** (critic 지적)
   - 발견자: critic
   - 위치: `world/world.json:terminology` vs 실제 사용
   - 문제:
     | 용어 | world.json | 실제 사용 (검색 결과) |
     |------|-----------|---------------------|
     | 마법사 칭호 | "메이지" | ch 3: "위저드" 발견 |
     | 화폐 단위 | "크라운" | ch 15: "크레딧" 발견 |
   - 영향: 독자 혼란, 일관성 훼손
   - 제안: 전체 텍스트에서 용어 통일 (자동 수정 가능)

#### 우선순위 낮음

5. **마법 학교 설정 미흡**
   - 발견자: genre-validator
   - 위치: `world/world.json:institutions`
   - 문제: 마법 학교가 언급만 되고 상세 정보 없음
   - 영향: 큰 문제는 아니지만, 플롯에 사용 시 추가 설정 필요
   - 제안: 나중에 필요할 때 추가해도 됨

## 자동 수정 옵션

일부 이슈는 자동으로 수정할 수 있습니다:

- ✅ 용어 불일치 ("위저드" → "메이지", "크레딧" → "크라운")
- ✅ JSON 구조 오류 (없음)

자동 수정하시겠습니까?

1. **자동 수정** - 즉시 수정하고 재검토
2. **직접 수정** - 수동으로 수정
3. **무시하고 진행** - 현재 상태로 진행 (권장하지 않음)

## Example 2: 캐릭터 검토

### 사용자 요청
```
사용자: 주인공 캐릭터 검토해줘
```

### 스킬 실행
```markdown
# 캐릭터 "유나" (char_001) 검토 결과

## 최종 판정: REVISE

**종합 점수**: 68/100
**신뢰도**: MEDIUM (편차: 18)

---

## 리뷰 요약

| 관점 | 점수 | 판정 | 주요 이슈 |
|------|------|------|-----------|
| 기술적 품질 (critic) | 75/100 | REVISE | 캐릭터 아크가 불명확 |
| 독자 경험 (beta-reader) | 58/100 | WEAK | 매력도 부족, 공감하기 어려움 |
| 장르 적합성 (genre-validator) | 72/100 | ACCEPTABLE | 로맨스 여주로는 약간 수동적 |

⚠️ **신뢰도 주의**: 리뷰 간 의견 차이가 있습니다.
- Critic은 "기술적으로는 괜찮다"
- Beta-Reader는 "독자가 좋아하지 않을 것"

---

## 상세 피드백

### ✅ 강점

#### 기술적 측면
1. **Want vs Need가 명확함**
   - 위치: `characters/char_001.json:want_need`
   - Want: "회사에서 승진하여 인정받기"
   - Need: "타인의 인정이 아닌 자기 가치 발견"
   - 평가: 외적 목표와 내적 필요의 충돌 구조가 좋음

2. **백스토리가 상세함**
   - 위치: `characters/char_001.json:backstory`
   - 어린 시절 부모의 기대, 학창 시절 왕따 경험
   - 현재 행동을 잘 설명함

#### 장르 측면
1. **능동적 성향**
   - 수동적 여주가 아닌 능동적으로 문제 해결
   - 현대 로맨스 독자 선호에 부합

---

### ❌ 문제점

#### 우선순위 최고 (치명적)

1. **캐릭터 아크가 불명확** (critic 지적)
   - 발견자: critic
   - 위치: `characters/char_001.json:arc`
   - 현재 내용:
     ```json
     {
       "arc": "자존감 회복과 진정한 사랑 발견"
     }
     ```
   - 문제: 너무 추상적. 구체적 단계가 없음
   - 영향: 집필 시 캐릭터 성장을 어떻게 표현할지 모호함
   - 제안:
     단계별 아크 설계:
     ```json
     {
       "arc": {
         "stage_1": "타인 인정에 집착 (ch 1-15)",
         "stage_2": "인정의 공허함 깨달음 (ch 16-30)",
         "stage_3": "자기 가치 탐색 (ch 31-40)",
         "stage_4": "진정한 자존감 확립 (ch 41-50)"
       }
     }
     ```

2. **매력도 부족** (beta-reader 지적)
   - 발견자: beta-reader
   - 위치: `characters/char_001.json` 전반
   - 문제: 독자가 "유나 응원하고 싶다!" 느낄 요소 부족
   - 구체적 이슈:
     - 특별한 강점 없음 (평범한 회사원)
     - 유머/위트 없음 (진지하기만 함)
     - 독특한 매력 포인트 없음
   - 영향: 독자가 "주인공이 재미없다" 느낄 위험 80%
   - 제안:
     매력 포인트 1-2개 추가:
     - 옵션 1: 독특한 취미 (예: 칵테일 믹싱, 타로 카드)
     - 옵션 2: 반전 매력 (겉으로는 완벽, 집에서는 게으름뱅이)
     - 옵션 3: 유머 센스 (자조적 유머로 스트레스 해소)

#### 우선순위 높음

3. **공감성 낮음** (beta-reader 지적)
   - 발견자: beta-reader
   - 위치: `characters/char_001.json:personality`
   - 문제:
     - Fear: "실패에 대한 두려움" (너무 일반적)
     - Lie: "인정받아야만 가치 있다" (공감하기 어려움)
   - 영향: 독자가 "나랑 다르다", "이해 안 된다" 느낌
   - 제안:
     더 구체적이고 공감 가능한 두려움:
     ```json
     {
       "fear": "혼자 남겨지는 것. 모두가 떠나고 나만 뒤처지는 상황",
       "lie": "완벽해야만 사랑받을 수 있다. 약점을 보이면 버려진다"
     }
     ```

4. **로맨스 여주로 약간 수동적** (genre-validator 지적)
   - 발견자: genre-validator
   - 위치: `characters/char_001.json:actions`
   - 문제:
     - 연애 관련 행동이 대부분 반응형 (남주가 주도, 유나는 응답)
     - 로맨스 장르에서는 여주도 능동적으로 사랑 추구해야 함
   - 영향: "남주 빨인 여주" 비판 가능성
   - 제안:
     유나가 먼저 행동하는 순간 추가:
     - 챕터 20: 유나가 먼저 데이트 제안
     - 챕터 35: 유나가 먼저 고백 (거절당해도)
     - 챕터 47: 유나가 주도적으로 갈등 해결
